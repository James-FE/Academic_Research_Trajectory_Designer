# Root Problem Framework

本文件是 Trajectory Designer v2 的核心。

它把张欢欢老师的选题框架转化为可操作的 LLM Deep Research 判断系统。

## 一句话原则

> 好博士方向不是热门方向，而是一个真实重要、长期存在、可深挖、可积累、能形成主线的根问题。

## 根问题的定义

根问题不是技术名词，不是社区标签，不是 benchmark 名字，不是模型类别。

根问题必须用这个句式表达：

> 在 ______ 真实场景中，面对 ______ 稳定约束 / 核心矛盾，如何实现 ______ 长期目标？

### 合格表达

> 在低资源科学研究中，面对数据稀缺、噪声高、实验反馈慢的问题，如何让机器学习模型产生可验证、可泛化的科学假设？

> 在公共健康监测中，面对隐私限制、跨地区分布偏移和标注不足，如何构建可靠、可解释、可迁移的早期预警模型？

> 在开放信息生态中，面对多源信息污染、立场操控和证据链断裂，如何让 AI 系统帮助人识别可信公共知识？

### 不合格表达

> 我做 LLM reasoning。  
> 我做 AI for science。  
> 我做多模态。  
> 我做可信 AI。  
> 我做医疗大模型。  
> 我做某个 benchmark 的 SOTA。

这些只是方向标签，不是根问题。

## 张老师四性过滤器

### 1. 可持续性

判断问题：

- 这个根问题未来 3–5 年还存在吗？
- 10 年后是否仍有研究价值？
- 它是否依赖短期热点、单一模型、单一 benchmark、单一数据集？
- 如果当前最热模型明年过时，这个根问题是否仍然成立？

通过信号：

- 背后是长期稳定的真实矛盾。
- 外部世界会持续制造这个问题。
- 模型换代只会改变方法，不会消灭问题。

淘汰信号：

- 根问题实际上是“如何在某 benchmark 上超过 SOTA”。
- 依赖某个短期社区 hype。
- 依赖某个马上会被大公司产品化吞掉的功能点。

### 2. 可深挖性

判断问题：

- 这个根问题能否自然长出 3–5 个递进子问题？
- 做完第一个问题之后，第二个问题是否会自然浮现？
- 这些子问题之间是因果 / 递进关系，还是为了凑 paper？

通过信号：

- 能形成问题树。
- 子问题之间有“测量 → 机制 → 方法 → 泛化 → 系统 / 影响”的自然关系，或有其他领域特有递进逻辑。
- 每个子问题都能成为一篇 paper 的核心。

淘汰信号：

- 只能想到一个 paper idea。
- 4 篇 paper 之间只是同一个模板换任务。
- 每篇 paper 都需要重新讲一个新故事。

### 3. 可积累性

判断问题：

- 第一篇 paper 建立的数据、代码、实验平台、理论视角，能否被后续 paper 复用？
- 后续工作是否能复用同一 evaluation substrate 或同一核心分析框架？
- 我的 expertise 是否会在这个方向上形成复利？

通过信号：

- 第 1 篇建立 measurement / baseline / failure taxonomy。
- 第 2 篇直接复用第 1 篇的发现和平台。
- 第 3 篇扩展到更复杂条件或更广泛场景。
- 第 4 篇做系统化整合或真实影响验证。

淘汰信号：

- 每篇 paper 都换数据集、换任务、换场景、换故事。
- 前一篇对后一篇没有认知资产贡献。
- 博士最后只能写“我做了 A/B/C/D 四件事”。

### 4. 主线感

判断问题：

- 我能否用一句话解释博士期间所有工作都在回答什么？
- 每篇 paper 是否都能挂回这个句子？
- 别人介绍我时，是否能说“他/她一直在解决那个问题”？

主线句模板：

> 我博士期间研究的是：在 ______ 场景中，面对 ______ 约束，如何实现 ______ 目标。

通过信号：

- 所有 paper 都能自然挂到主线句。
- 主线句不是空泛领域标签，而是一个明确问题。
- 主线句里包含真实场景、稳定约束、长期目标。

淘汰信号：

- 主线必须绕很多弯才能解释。
- 主线只是“我做 AI for X”。
- 论文之间只能靠领域标签拼接。

## 根问题树模板

一个好的根问题通常可以长成这样：

```text
根问题：在 [真实场景] 中，面对 [稳定约束]，如何实现 [长期目标]？

├── P1：这个问题如何被定义、测量、诊断？
│   └── 可能产出：measurement / benchmark analysis / failure taxonomy / baseline audit
├── P2：当前方法为什么失败？关键机制或瓶颈是什么？
│   └── 可能产出：causal analysis / robustness study / theoretical characterization
├── P3：如何解决第一个核心瓶颈？
│   └── 可能产出：method / model / algorithm / training objective / system mechanism
├── P4：在更复杂、更真实、更分布外的环境中如何保持有效？
│   └── 可能产出：generalization / adaptation / uncertainty / transfer / scaling
└── P5：如何集成成可复用系统或形成真实影响？
    └── 可能产出：system / toolkit / field validation / policy-relevant evaluation
```

注意：这只是骨架，不是固定模板。不同领域可以有不同递进逻辑。  
如果所有候选方向都长成完全一样的问题树，说明你在套模板。

## 根问题评分卡

每项 0–5 分。低于阈值直接淘汰。

| 维度 | 解释 | 0 分 | 5 分 |
|---|---|---|---|
| 真实重要性 | 是否是真实世界 / 科学 / 公共利益中的痛点 | 伪问题 | 长期痛点 |
| 稳定核心矛盾 | 是否不依赖短期模型或 benchmark | 热点依赖 | 长期稳定 |
| 可持续性 | 3–5 年是否仍有空间 | 很快过时 | 长期生长 |
| 可深挖性 | 是否能长出问题树 | 只有一个点 | 3–5 个自然节点 |
| 可积累性 | 平台 / 数据 / 认知是否复用 | 每篇重来 | 复利明显 |
| 主线感 | 是否一句话讲清楚 | 领域标签 | 清晰根问题 |
| Evaluation substrate | 是否已有公开评估基础 | 需要自造 | 数据 / baseline / metric 可复用 |
| 非对称困难 | 是否适合单兵博士切入 | 大资源碾压 | 单兵有杠杆 |
| 3 个月 entry | 第一篇能否启动 | 无法启动 | 有明确 baseline |
| 4-paper narrative | 是否支撑博士论文 | 拼盘 | 自然递进 |

### Go / Hold / Kill 规则

Go：

- 总分 ≥ 38 / 50
- 主线感 ≥ 4
- 可深挖性 ≥ 4
- Evaluation substrate ≥ 4
- 3 个月 entry ≥ 4

Hold：

- 总分 30–37
- 但缺口可以在一周内补证据
- 或者根问题强，但可执行性需要再查证

Kill：

- 主线感 < 3
- 可深挖性 < 3
- Evaluation substrate < 3
- 3 个月 entry < 3
- 明显违反 value axis
- 需要长期数据谈判
- 只能靠 synthetic data 自说自话
- 4 篇 paper 明显是凑的

## 反模式清单

### 反模式 1：方向标签伪装成根问题

坏表达：

> 我想做 AI for climate。

追问：

> 哪个真实场景？什么约束？什么长期目标？谁受益？为什么现在没解决？

### 反模式 2：方法机会伪装成根问题

坏表达：

> 我想用 LLM / diffusion / graph neural network / RL 解决 X。

追问：

> 如果不用这个方法，根问题还存在吗？  
> 如果明年这个方法过时，你的博士主线还成立吗？

### 反模式 3：benchmark race 伪装成根问题

坏表达：

> 这个 benchmark 还没人刷到很高。

追问：

> benchmark 背后对应什么真实问题？  
> SOTA 提升是否真的推进了根问题？

### 反模式 4：蓝海幻想

坏表达：

> 没人做，所以是蓝海。

追问：

> 为什么没人做？是被低估，还是数据不可得、问题不重要、论文发不出？

### 反模式 5：四篇 paper 拼盘

坏表达：

> Paper 1 做数据集，Paper 2 做模型，Paper 3 做解释，Paper 4 做应用。

追问：

> 为什么必须是这个顺序？  
> Paper 2 解决的是 Paper 1 暴露出的哪个问题？  
> Paper 3 为什么不是另一个独立故事？

## 最终判断

一个根问题只有在你能清晰回答下面这句话时，才进入博士主线候选池：

> 这个根问题真实存在，未来 3–5 年不会消失，我能在 3 个月内启动第一篇 paper，并且它能自然长出一棵问题树，支撑 4 篇互相递进、可积累、有主线感的论文。
