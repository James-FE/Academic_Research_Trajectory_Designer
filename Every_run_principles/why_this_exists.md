# Why This Exists

我需要形成我的研究方向感，而不是迷失在论文的海洋里。

但“研究方向感”还不是终点。真正要找的是：

> 一个真实重要、长期存在、可深挖、可积累、能形成博士主线的根问题。

Trajectory Designer v2 存在的原因，是因为博士阶段的主线选择不能由某个 paper corpus、某个 benchmark trend、某个热门社区、某个短期投稿机会，或者“我现在会什么技术”来决定。

我博士阶段会什么并不重要，我可以学。重要的是：

> 我可以去解决什么？  
> 我要长期回答什么根问题？  
> 这个根问题是否值得我投入 3–5 年？

## 从“方向”前移到“根问题”

方向标签通常长这样：

- AI for science
- LLM reasoning
- multimodal learning
- AI for healthcare
- scientific machine learning
- trustworthy AI
- embodied AI

这些不是根问题。它们只是领域标签。

根问题必须长这样：

> 在 ______ 真实场景中，面对 ______ 稳定约束 / 核心矛盾，如何实现 ______ 长期目标？

例如：

> 在真实动态网络环境中，面对带宽波动、拥塞、丢包和多用户竞争，如何实现低延迟、高可靠的视频传输？

只有这样的表达，才可能自然长出问题树。

## 本系统要避免什么

Trajectory Designer v2 不为以下事情服务：

- 找热门 topic。
- 找最容易发 paper 的 benchmark。
- 给我 10 个看起来不错但不可执行的宏观方向。
- 让我用 LLM 的宏观视野强化自己的幻想。
- 把博士选题变成漂亮表格。
- 把战略判断逃避成 prompt engineering。
- 把“蓝海”误解为没有 benchmark / dataset / metric 的荒地。
- 把“我会什么”误认为“我应该做什么”。

## 本系统要完成什么

Trajectory Designer v2 要逼迫我完成四个判断：

1. **根问题判断**：这个问题是否真实、长期、重要？
2. **问题树判断**：它是否能自然长出 3–5 个递进子问题？
3. **可积累判断**：我的数据、代码、实验平台、理论视角是否能复用？
4. **主线判断**：我能否用一句话解释博士期间所有工作在回答什么？

## Benchmark 的正确位置

benchmark 不能决定我的长期方向。

但已有 evaluation substrate 是博士早期可执行性的硬门槛。

我不追逐 leaderboard，但我也不承担从零建设 task / metric / dataset 的基础设施债。

最理想的方向是：

> 已有 benchmark / dataset / metric / baseline，但 benchmark 社区还没有充分回答背后的真实科学 / 社会 / 公共问题。

## 每周快速试错的目的

一周一个新方向，不是为了“浅尝辄止”。

它的目标是尽早发现：

- 这个方向有没有根问题？
- 根问题是否真实？
- 是否有公开 evaluation substrate？
- 我是否有独特切入优势？
- 它能不能长出问题树？
- 它是否能支撑 4 篇递进 paper？
- 第一篇能否在 3 个月内启动？

一周试错结束后，必须产生一个明确判断：

- Go
- Hold
- Kill

没有判断，就不是试错，只是阅读。

## 最终输出

最终输出不是“方向列表”，而是一份：

- `trajectory_final.md`

它必须让我获得方向感，而不是把我带入另一片更大的论文海洋。

它必须回答：

> 我博士阶段的根问题是什么？  
> 我的第一篇 paper 为什么必须从这里开始？  
> 我的 4 篇 paper 如何自然递进？  
> 为什么这个方向符合我的 value axis？  
> 为什么它不是热点追逐、死海、散点 paper 或伪主线？
